A little background



ThousandEyes WAN Insights uses raw telemetry data to continuously observe the behavior of every network path. The biggest offering of WAN insights is our concept of Quality, which is a parameter we build by observing networks over time, and noticing the thresholds at which SLA violations take place. For example, if we are talking about the San Jose site between March 13 and March 20th, and we observe that loss was above the threshold of 3%, that would bring down the quality. So instead of having to monitor a lot of different aspects like loss, latency and jitter at all times, the network operations engineers can just observe quality.

The system also makes continuous recommendations for improving quality, for example, switch to blue-internet instead of bronze-internet to improve quality by 5%. This is our leading selling point.

However, when I was testing out these recommendations post GA, I heard a repeated concern from customers. They were worried about weather accepting the recommendation would mess up with the load balancing and whether the recommended path would still be ok to support the resulting traffic. This problem was huge because a) network changes could be expensive in some parts of the world. b) network changes often need to be approved by several personnel, which could be time consuming as much as 18 months. c) if the digital user experience of the end users was worse than before, that would mean losing business.
